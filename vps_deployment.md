# Инструкция по деплою ADR платформы на Timeweb Cloud

Эта инструкция поможет вам развернуть проект на чистом VPS сервере с использованием Docker.

## 1. Подготовка VPS (Timeweb Cloud)

1. Зайдите в панель [Timeweb Cloud](https://timeweb.cloud/).
2. Создайте новый облачный сервер:
   - **ОС**: Ubuntu 22.04
   - **Регион**: Любой (рекомендуется Москва/Санкт-Петербург)
   - **Конфигурация**: Минимально 1 CPU, 2 ГБ RAM (тариф «HighFreq» или стандартный).
3. После создания вы получите IP-адрес сервера и пароль root (придет на почту).

## 2. Установка Docker на сервер

Подключитесь к серверу через терминал (SSH):
```bash
ssh root@ВАШ_IP_АДРЕС
```

Выполните команды для установки Docker и Docker Compose:
```bash
# Обновление пакетов
apt update && apt upgrade -y

# Установка Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# Установка Docker Compose
apt install -y docker-compose
```

## 3. Загрузка проекта на сервер

Самый простой способ — через Git. Если репозиторий приватный, добавьте SSH-ключ сервера в GitHub/GitLab.

```bash
cd /root
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ> dopog
cd dopog
```

## 4. Настройка переменных окружения

Создайте файл `.env` на сервере:
```bash
nano .env
```

Вставьте туда настройки (замените `ВАШ_IP_АДРЕС` на реальный IP):
```env
AUTH_SECRET=9f8e7d6c5b4a32109f8e7d6c5b4a3210
NEXTAUTH_URL=http://ВАШ_IP_АДРЕС:3000
AUTH_TRUST_HOST=true
```
*(Нажмите `Ctrl+O`, `Enter`, `Ctrl+X` для сохранения и выхода)*

## 5. Подготовка базы данных

Чтобы проект начал работу, нужно создать папку для базы данных и (опционально) перенести существующую базу:
```bash
mkdir -p data
# Если хотите перенести текущие вопросы с локального ПК:
# Просто скопируйте dev.db в папку data/prod.db на сервере (через FileZilla или scp)
```

## 6. Запуск проекта

Соберите и запустите контейнер:
```bash
docker-compose up -d --build
```

После завершения билда сайт будет доступен по адресу: `http://ВАШ_IP_АДРЕС:3000`.

---

## 7. Привязка домена (Когда купите домен)

Когда у вас появится домен (например, `adr-exam.ru`):
1. В панели Timeweb направьте `A-запись` домена на IP сервера.
2. Обновите `NEXTAUTH_URL` в файле `.env` на сервере:
   ```env
   NEXTAUTH_URL=https://adr-exam.ru
   ```
3. Рекомендуется установить **Nginx** и **Certbot** (SSL) для работы по HTTPS. Если нужно, я дам инструкцию и для этого шага.

> [!TIP]
> Команда `docker-compose logs -f app` поможет вам увидеть логи приложения в реальном времени, если что-то пойдет не так.
